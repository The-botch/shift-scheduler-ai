設計ポイント:

  • • 左サイドメニュー: 年月選択（2024年11月、2024年12月、2025年1月...）
  • • 対象月のみ緑色でアクティブ、他はグレーアウト＆ロックアイコン
  • • メインエリア: 3つのカード（募集状況、第一案、第二案）
  • • 募集状況カード → モニタリング へ遷移（希望提出状況確認）
  • • 第一案カード → FirstPlanEditor へ遷移（作成/編集/確認）
  • • 第二案カード → SecondPlanEditor へ遷移（作成/編集/確認）
  • • 第一案が未承認の場合、第二案はグレーアウト
  • • 締切日: 内部固定（前月20日）、将来的に設定可能化
  • • 承認後/確定後も編集可能（ロック機能は実装しない）
  • • PDF出力: トップ画面には配置しない。SecondPlanEditor内のCSVボタン横に配置（Issue #167）

対象月切り替えロジック:

  • • 20日以前: 来月が対象月
  • • 21日以降: 再来月が対象月
  • • 例: 11/28 → 2025年1月が対象月
  • • 例: 12/15 → 2025年1月が対象月
  • • 例: 12/21 → 2025年2月が対象月

対象月の判定ロジック

const getTargetMonth = () => {
  const now = new Date()
  const currentDay = now.getDate()
  const currentMonth = now.getMonth() + 1
  const currentYear = now.getFullYear()

  // 20日以前：来月が対象
  // 21日以降：再来月が対象
  if (currentDay <= 20) {
    return {
      year: currentMonth === 12 ? currentYear + 1 : currentYear,
      month: currentMonth === 12 ? 1 : currentMonth + 1
    }
  } else {
    const targetMonth = currentMonth + 2
    return {
      year: targetMonth > 12 ? currentYear + 1 : currentYear,
      month: targetMonth > 12 ? targetMonth - 12 : targetMonth
    }
  }
}

const isMonthAccessible = (year, month) => {
  const target = getTargetMonth()
  return year === target.year && month === target.month
}

const getMonthStatus = async (year, month) => {
  const plan = await getShiftPlan(storeId, year, month)

  if (!plan) return 'not_started'
  if (plan.plan_type === 'SECOND' && plan.status === 'APPROVED')
    return 'completed'
  if (plan.plan_type === 'FIRST' && plan.status === 'APPROVED')
    return 'first_approved'
  if (plan.status === 'DRAFT')
    return 'draft'

  return 'not_started'
}
