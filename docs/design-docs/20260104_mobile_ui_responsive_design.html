作成日: 2026-01-04

ステータス: 計画中

対象ブランチ: feat/mobile-ui-responsive

1. 背景・目的

1.1 現状の課題

現在のシフト管理Webアプリケーションは、PC向けに設計されており、モバイルデバイスでの使用時に以下の問題が発生している：

  • 画面が崩れて操作不能になる
  • サイドバーが固定幅で画面を占有
  • テーブルが横スクロールなしで途切れる
  • ボタンやフォーム要素が小さすぎてタップ困難

1.2 目的

緊急時にスマートフォンからでもシフト管理操作（確認・編集）を可能にする。

決定事項:
  • シフト作成画面は「閲覧のみ」ではなく「編集も完全対応」
  • iPadはPC版と同じレイアウト（640px以上）
  • 分割表示ではなく「スライド表示」（オフキャンバス）を採用

2. 現状分析

画面 現状対応度 主な問題点 優先度
ダッシュボード (ShiftDashboard) 32% サイドバー固定幅288px、3カラム固定グリッド 高
スタッフ一覧 (StaffManagement) 40% テーブル8カラム、フィルター1行詰め込み 中
希望一覧 (Monitoring) 70% 比較的良好、月ボタン・サマリーカードの調整のみ 低
シフト作成画面 (FirstPlanEditor/SecondPlanEditor) 10% 最小幅1200px要求、完全デスクトップ専用設計 最高

3. 設計方針

3.1 ブレークポイント戦略

デバイス 画面幅 レイアウト Tailwind プレフィックス
スマートフォン < 640px モバイルUI（スライドメニュー、縦積み） デフォルト（プレフィックスなし）
iPad / タブレット / PC >= 640px PC版UI（サイドバー常時表示、横スクロール） sm:
重要: 分岐点は sm: (640px) を使用。これにより、PC版のレイアウトには一切影響を与えずにモバイル対応を追加できる。

3.2 設計原則

原則 説明 実装方法
スライド表示 サイドバーをオフキャンバス化（モバイルのみ） fixed + translate-x + transition
PC版への影響なし sm:プレフィックスで分岐 モバイル向けスタイルをデフォルトに、PC向けをsm:で上書き
横スクロール維持 テーブルはカード変換せず横スクロール overflow-x-auto + 最小幅設定
ハンバーガーメニュー モバイルでサイドバー開閉 Lucide Menu / X アイコン使用

  overflow-y-auto

  {/* 編集フォーム */}
</div>

5. 共通CSS追加

frontend/src/index.css
/* モバイル対応用ユーティリティ */
@layer utilities {
  /* タッチデバイス用のタップターゲットサイズ確保 */
  .touch-target {
    @apply min-h-[44px] min-w-[44px];
  }

  /* モバイルでのオーバーフロー制御 */
  .mobile-scroll {
    @apply overflow-x-auto -mx-4 px-4;
    -webkit-overflow-scrolling: touch;
  }

  /* スクロールヒント（横スクロール可能を示す影） */
  .scroll-hint {
    background:
      linear-gradient(to right, white 30%, transparent),
      linear-gradient(to left, white 30%, transparent),
      linear-gradient(to right, rgba(0,0,0,0.1), transparent),
      linear-gradient(to left, rgba(0,0,0,0.1), transparent);
    background-position: left center, right center, left center, right center;
    background-size: 20px 100%, 20px 100%, 10px 100%, 10px 100%;
    background-repeat: no-repeat;
    background-attachment: local, local, scroll, scroll;
  }
}

8. リスクと対策

リスク 影響度 対策
PC版レイアウトへの意図しない影響 高 sm:プレフィックスを徹底使用、PC版での回帰テスト必須
シフト編集のタッチ操作が直感的でない 中 ユーザーテストを実施、操作ガイドを追加
テーブル横スクロールのUXが悪い 中 スクロールヒント（影）を追加、sticky列で現在位置を把握しやすく
パフォーマンス低下（リフロー増加） 低 CSSのみで対応、JavaScript追加は最小限に
