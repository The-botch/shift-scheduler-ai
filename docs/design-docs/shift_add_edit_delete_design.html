シフト追加・編集・削除機能 設計書

1. 概要

SecondPlanEditorにおいて、シフトセルをクリックすることで、シフトの追加・編集・削除ができる機能を実装する。

重要な設計原則:
  • 共通コンポーネントの使用: 新規追加と編集は同一のモーダルコンポーネントを使用する
  • 空のセルをクリック → 新規追加モード（フォームが空の状態）
  • 既存シフトをクリック → 編集モード（フォームに現在の値が入力済み）
  • 下書き保存機能との統合: 変更は即座にDBに保存せず、メモリに保持し、「下書き保存」ボタンで一括保存

4. コンポーネント設計

4.1 共通モーダルコンポーネント（ShiftEditModal）

設計の核心: 新規追加と編集で同じコンポーネントを使用することで、コードの重複を避け、保守性を向上させる。 
// SecondPlanEditor.jsx 内に定義

const ShiftEditModal = ({
  isOpen,           // モーダル表示フラグ
  onClose,          // 閉じる関数
  mode,             // 'add' | 'edit'
  shift,            // 編集モードの場合は既存シフトデータ、新規の場合は基本情報のみ
  onSave,           // 保存時のコールバック
  onDelete,         // 削除時のコールバック（編集モードのみ）
  preferences       // 希望シフトデータ（NG日チェック用）
}) => {
  const [startTime, setStartTime] = useState(shift?.start_time || '')
  const [endTime, setEndTime] = useState(shift?.end_time || '')
  const [breakMinutes, setBreakMinutes] = useState(shift?.break_minutes || 0)

  // 希望シフトのチェック
  const checkPreference = () => {
    const pref = preferences.find(p => parseInt(p.staff_id) === parseInt(shift.staff_id))
    if (!pref) return null

    const dateStr = shift.date // "2024-11-29" 形式

    // NG日チェック
    if (pref.ng_days) {
      const ngDays = pref.ng_days.split(',').map(d => d.trim())
      if (ngDays.includes(dateStr)) {
        return 'ng'
      }
    }

    // 希望日チェック
    if (pref.preferred_days) {
      const preferredDays = pref.preferred_days.split(',').map(d => d.trim())
      if (preferredDays.includes(dateStr)) {
        return 'preferred'
      }
    }

    return null
  }

  const handleSave = () => {
    // 必須項目チェック
    if (!startTime || !endTime) {
      alert('開始時刻と終了時刻を入力してください')
      return
    }

    // 時刻の妥当性チェック
    if (startTime >= endTime) {
      alert('終了時刻は開始時刻より後にしてください')
      return
    }

    // 希望シフトチェック
    const prefStatus = checkPreference()
    if (prefStatus === 'ng') {
      const confirmMsg = mode === 'add'
        ? 'この日はスタッフのNG日として登録されています。\n本当にシフトを追加しますか？'
        : 'この日はスタッフのNG日として登録されています。\n本当に更新しますか？'
      if (!confirm(confirmMsg)) {
        return
      }
    }

    onSave({
      start_time: startTime,
      end_time: endTime,
      break_minutes: parseInt(breakMinutes) || 0
    })
  }

  return (
    <AnimatePresence>
      {isOpen && (
        <motion.div className="modal-overlay">
          <div className="modal-content">
            <h2>{mode === 'add' ? 'シフト追加' : 'シフト編集'}</h2>

            {/* スタッフ・日付・店舗情報の表示 */}
            <div className="shift-info">
              <p>スタッフ: {shift.staff_name}</p>
              <p>日付: {shift.date}</p>
              <p>店舗: {shift.store_name}</p>
            </div>

            {/* 時刻入力フォーム */}
            <div>
              <label>開始時刻 *</label>
              <input
                type="time"
                value={startTime}
                onChange={(e) => setStartTime(e.target.value)}
              />
            </div>
            <div>
              <label>終了時刻 *</label>
              <input
                type="time"
                value={endTime}
                onChange={(e) => setEndTime(e.target.value)}
              />
            </div>
            <div>
              <label>休憩時間（分）</label>
              <input
                type="number"
                value={breakMinutes}
                onChange={(e) => setBreakMinutes(e.target.value)}
                min="0"
                step="15"
                placeholder="例: 60"
              />
            </div>

            {/* ボタン群 */}
            <div className="button-group">
              {mode === 'edit' && (
                <button onClick={onDelete} className="button-danger">
                  削除
                </button>
              )}
              <button onClick={onClose} className="button-secondary">
                キャンセル
              </button>
              <button onClick={handleSave} className="button-primary">
                {mode === 'add' ? '追加' : '更新'}
              </button>
            </div>
          </div>
        </motion.div>
      )}
    </AnimatePresence>
  )
}

const handleCellClick = (date, staffId, storeId) => {
  // 既存シフトがあるか確認
  const existingShift = getShiftForCell(date, staffId, storeId)

  if (existingShift) {
    // 編集モード
    onShiftClick?.({
      mode: 'edit',
      shift: existingShift,
      date,
      staffId,
      storeId
    })
  } else {
    // 新規追加モード
    onShiftClick?.({
      mode: 'add',
      shift: {
        date,
        staff_id: staffId,
        store_id: storeId,
        staff_name: getStaffName(staffId),
        store_name: getStoreName(storeId)
      }
    })
  }
}

4.3 SecondPlanEditorでの統合

// SecondPlanEditor.jsx

const [modalState, setModalState] = useState({
  isOpen: false,
  mode: 'add',  // 'add' | 'edit'
  shift: null
})

// セルクリック時のハンドラー
const handleShiftClick = ({ mode, shift }) => {
  setModalState({
    isOpen: true,
    mode,
    shift
  })
}

// モーダルからの保存処理
const handleModalSave = (timeData) => {
  if (modalState.mode === 'add') {
    handleAddShift({
      ...modalState.shift,
      ...timeData
    })
  } else {
    handleUpdateShift(modalState.shift.shift_id, timeData)
  }
  setModalState({ isOpen: false, mode: 'add', shift: null })
}

// モーダルからの削除処理
const handleModalDelete = () => {
  if (!confirm('このシフトを削除しますか？')) return
  handleDeleteShift(modalState.shift.shift_id)
  setModalState({ isOpen: false, mode: 'add', shift: null })
}

return (
  <>
    {/* シフト表 */}
    <MultiStoreShiftTable
      onShiftClick={handleShiftClick}
      // ... other props
    />

    {/* 共通モーダル */}
    <ShiftEditModal
      isOpen={modalState.isOpen}
      mode={modalState.mode}
      shift={modalState.shift}
      preferences={preferences}
      onClose={() => setModalState({ isOpen: false, mode: 'add', shift: null })}
      onSave={handleModalSave}
      onDelete={handleModalDelete}
    />
  </>
)

5. 状態管理

状態変数 型 説明
modalState Object モーダルの表示状態、モード（add/edit）、対象シフトデータを保持
addedShifts Array 新規追加されたシフトのリスト（下書き保存時にDBに追加）
modifiedShifts Object 編集されたシフトのマップ（shift_id → 変更データ）
deletedShiftIds Set 削除対象のshift_idセット
hasUnsavedChanges Boolean 未保存の変更があるかのフラグ

7. 注意事項

仮IDの扱い

新規追加されたシフトは、shift_idがまだDBで割り当てられていないため、仮IDを使用する:

const tempId = `temp_${Date.now()}_${Math.random()}`

const newShift = {
  shift_id: tempId,  // 仮ID
  staff_id: staffId,
  store_id: storeId,
  date: date,
  start_time: startTime,
  end_time: endTime,
  break_minutes: breakMinutes,
  modified_flag: true
}

データ整合性

  • csvShifts, dayShifts, addedShifts の3つの状態を常に同期させる
  • 削除されたシフトはcsvShifts/dayShiftsから削除し、deletedShiftIdsに追加
  • 編集されたシフトはcsvShifts/dayShiftsを即座に更新し、modifiedShiftsにも記録

エラーハンドリング

  • 時刻の妥当性チェック: 開始時刻 < 終了時刻
  • 必須項目の入力チェック: 開始時刻と終了時刻は必須
  • 休憩時間の妥当性チェック: 0以上の整数、勤務時間を超えないこと
  • 希望シフトチェック:
    • NG日にシフトを追加/更新する場合 → 警告ダイアログを表示
    • 希望日にシフトを追加する場合 → 通常通り処理（警告なし）
  • 保存失敗時のロールバック処理

コンポーネント再利用のメリット

  • ✅ コードの重複を避けられる（DRY原則）
  • ✅ バグ修正やメンテナンスが1箇所で済む
  • ✅ ユーザー体験が統一される（同じUIで追加と編集ができる）
  • ✅ テストが容易になる（1つのコンポーネントのみテストすればよい）

8. まとめ

本設計では、ShiftEditModalという1つの共通コンポーネントを使用して、シフトの新規追加と編集の両方を実現します。

主な特徴:

  • 共通コンポーネント: mode propsによって動作を切り替え、編集モードの場合のみ削除ボタンを表示
  • 必要な入力項目: 開始時刻、終了時刻、休憩時間（分）
  • バリデーション機能:
    • 必須項目チェック
    • 時刻の妥当性チェック（開始 < 終了）
    • 休憩時間の妥当性チェック
    • 希望シフトチェック: NG日に登録・更新しようとする場合は警告ダイアログを表示
  • 下書き保存との統合: 複数の変更を行った後に一括で保存できる柔軟な操作性

この設計により、シンプルかつ保守性の高い実装を実現し、ユーザーにとって使いやすいシフト管理機能を提供します。
